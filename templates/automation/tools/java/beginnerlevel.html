4. Control Statements

Control statements in Java help us decide the flow of execution. They determine which block of code runs based on conditions.

    }

Explanation: The switch statement selects one block of code to run based on the matching case value.

11. The this Keyword

The this keyword refers to the current object inside a class. It helps differentiate between instance variables and parameters when they share the same name.

ðŸ’» Example: Using 'this'


class Employee {
    String name;
    int salary;

    Employee(String name, int salary) {
        this.name = name;       // using 'this' to refer to instance variables
        this.salary = salary;
    }

    void showDetails() {
        System.out.println("Employee: " + name + ", Salary: " + salary);
    }
}

public class ThisKeywordExample {
    public static void main(String[] args) {
        Employee emp1 = new Employee("Ravi", 50000);
        emp1.showDetails();
    }
}

Explanation: The this keyword removes ambiguity between instance and local variables with the same name.

12. Static Members in Java

The static keyword is used for variables and methods that belong to the class, not to any specific object. They can be accessed directly using the class name.

ðŸ’» Example: Static Variables and Methods


class Counter {
    static int count = 0;

    Counter() {
        count++;
        System.out.println("Object created. Count = " + count);
    }

    static void showCount() {
        System.out.println("Total objects created: " + count);
    }
}

public class StaticExample {
    public static void main(String[] args) {
        new Counter();
        new Counter();
        Counter.showCount(); // called using class name
    }
}

Explanation: Static members are shared across all objects of a class. In this example, the counter increases every time a new object is created.

13. Access Modifiers and Encapsulation

Access modifiers control the visibility of classes, variables, and methods. Encapsulation binds data and methods together and restricts direct access to internal data.

  • public â€“ accessible from anywhere.
  • private â€“ accessible only within the same class.
  • protected â€“ accessible within package and subclasses.
  • default â€“ accessible only within the same package.

ðŸ’» Example: Encapsulation


class Account {
    private int balance = 10000;

    // getter method
    public int getBalance() {
        return balance;
    }

    // setter method
    public void deposit(int amount) {
        balance += amount;
    }
}

public class EncapsulationExample {
    public static void main(String[] args) {
        Account acc = new Account();
        System.out.println("Initial Balance: " + acc.getBalance());
        acc.deposit(5000);
        System.out.println("Updated Balance: " + acc.getBalance());
    }
}

Explanation: The balance field is private and can only be modified through public methods. This protects sensitive data and demonstrates encapsulation.

Introduction to Object-Oriented Programming (OOPs)

Java is an Object-Oriented Programming Language. OOPs concepts help organize code into objects and classes, improving maintainability and reusability.

  • Class: Blueprint for creating objects.
  • Object: Instance of a class.
  • Encapsulation: Binding data and methods together.
  • Inheritance: Acquiring properties of another class.
  • Polymorphism: One interface, multiple implementations.
  • Abstraction: Hiding internal details and showing essential features.
