1. Advanced OOP Concepts

Object-Oriented Programming in Java is the core principle where programs are organized around objects rather than actions. Let's explore advanced topics like inheritance, polymorphism, abstraction, and interfaces.

ðŸ‘‰ Inheritance

Inheritance allows one class to acquire the properties and behaviors of another class using the extends keyword. This promotes code reuse.


class Vehicle {
    void start() {
        System.out.println("Vehicle is starting...");
    }
}

class Car extends Vehicle {
    void honk() {
        System.out.println("Car is honking!");
    }
}

public class Main {
    public static void main(String[] args) {
        Car myCar = new Car();
        myCar.start(); // inherited method
        myCar.honk();  // subclass method
    }
}

ðŸ‘‰ Method Overriding

When a subclass defines a method that already exists in the parent class with the same signature, it's called overriding. The subclass method is executed at runtime.


class Animal {
    void sound() {
        System.out.println("Animal makes sound");
    }
}

class Dog extends Animal {
    @Override
    void sound() {
        System.out.println("Dog barks");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal a = new Dog(); // Polymorphic reference
        a.sound(); // Output: Dog barks
    }
}

ðŸ‘‰ Abstraction

Abstraction is the process of hiding implementation details and showing only functionality. This is achieved using abstract classes and interfaces.


abstract class Shape {
    abstract void draw(); // abstract method
}

class Circle extends Shape {
    void draw() {
        System.out.println("Drawing a Circle");
    }
}

public class Main {
    public static void main(String[] args) {
        Shape s = new Circle();
        s.draw(); // Output: Drawing a Circle
    }
}

ðŸ‘‰ Interfaces

An interface defines a contract that classes must implement. A key benefit is that a class can implement multiple interfaces.


interface Flyable {
    void fly();
}

interface Swimmable {
    void swim();
}

class Duck implements Flyable, Swimmable {
    public void fly() {
        System.out.println("Duck is flying");
    }
    public void swim() {
        System.out.println("Duck is swimming");
    }
}

public class Main {
    public static void main(String[] args) {
        Duck d = new Duck();
        d.fly();
        d.swim();
    }
}

2. Exception Handling

Exceptions are runtime errors that disrupt normal program execution. Java provides a structured way to handle exceptions using checked and unchecked exceptions, and the `try`, `catch`, `finally` block.

ðŸ‘‰ Tryâ€“Catchâ€“Finally


public class MainException {
    public static void main(String[] args) {
        try {
            int result = 10 / 0;  // ArithmeticException (Unchecked)
            System.out.println(result);
        } catch (ArithmeticException e) {
            System.out.println("Cannot divide by zero!");
        } finally {
            // Executes regardless of whether an exception occurred
            System.out.println("Finally block always executes.");
        }
    }
}

ðŸ‘‰ Throw and Throws

The `throw` keyword manually throws an exception. The `throws` keyword declares that a method might throw an exception, forcing the caller to handle it.


class AgeCheck {
    static void verifyAge(int age) throws Exception {
        if (age < 18)
            // throw keyword
            throw new Exception("Underage not allowed! (Checked Exception)");
        else
            System.out.println("Access granted.");
        }

    public static void main(String[] args) {
        try {
            verifyAge(15);
        } catch (Exception e) {
            System.out.println("Caught: " + e.getMessage());
        }
    }
}

3. Packages & Access Modifiers

Packages group related classes and interfaces, preventing naming conflicts. Access modifiers control visibility across classes and packages.

ðŸ‘‰ Access Modifiers

Java provides 4 levels of access control: public, protected, default (no keyword), and private.


public class ExampleAccess {
    // Accessible everywhere
    public int a = 10;
    // Accessible within package and by subclasses
    protected int b = 20;
    // Accessible only within the package (default)
    int c = 30;
    // Accessible only within this class
    private int d = 40;

    public void show() {
        // All are accessible inside the class
        System.out.println("a=" + a + ", b=" + b + ", c=" + c + ", d=" + d);
    }

    public static void main(String[] args) {
        new ExampleAccess().show();
    }
}

4. Inner Classes

Inner classes are defined within another class. They logically group classes and can access the outer class's members, including private ones.

ðŸ‘‰ Member Inner Class


class Outer {
    private String message = "Hello from Inner Class";

    class Inner {
        void display() {
            // Can access private member of Outer
            System.out.println(message);
        }
    }

    public static void main(String[] args) {
        Outer outer = new Outer();
        // Instance of Inner requires an instance of Outer
        Outer.Inner inner = outer.new Inner();
        inner.display();
    }
}

ðŸ‘‰ Anonymous Inner Class

Used for implementing an interface or abstract class on the fly, typically when you need a single-use object.


abstract class Greeting {
    abstract void sayHello();
}

public class MainAnonymous {
    public static void main(String[] args) {
        // Defines a subclass of Greeting and instantiates it immediately
        Greeting g = new Greeting() {
            void sayHello() {
                System.out.println("Hello from Anonymous Class");
            }
        };
        g.sayHello();
    }
}

5. Wrapper Classes

Wrapper classes convert primitive data types (like `int`, `char`) into objects (like `Integer`, `Character`). This is crucial for using primitives in Java Collections.

ðŸ‘‰ Autoboxing and Unboxing

Autoboxing is the automatic conversion of a primitive to its wrapper object. Unboxing is the reverse. 


public class MainWrapper {
    public static void main(String[] args) {
        int a = 10;
        Integer obj = a;  // Autoboxing (int -> Integer)
        int b = obj;      // Unboxing (Integer -> int)

        System.out.println("Primitive b: " + b);
        System.out.println("Object obj: " + obj);
    }
}

6. Collections Framework

The Java Collections Framework provides classes and interfaces to store and manipulate groups of objects efficiently, primarily centered around List, Set, and Map.

ðŸ‘‰ List (ArrayList Example)


import java.util.List;
import java.util.ArrayList;

public class MainList {
    public static void main(String[] args) {
        // Ordered, allows duplicates
        List<String> list = new ArrayList<>();
        list.add("Java");
        list.add("Python");
        list.add("Python");

        System.out.println("List: " + list);
    }
}

ðŸ‘‰ Set (HashSet Example)


import java.util.Set;
import java.util.HashSet;

public class MainSet {
    public static void main(String[] args) {
        // Unordered, does NOT allow duplicates
        Set<Integer> numbers = new HashSet<>();
        numbers.add(10);
        numbers.add(20);
        numbers.add(10); // Ignored

        System.out.println("Set: " + numbers);
    }
}

ðŸ‘‰ Map (HashMap Example)


import java.util.Map;
import java.util.HashMap;

public class MainMap {
    public static void main(String[] args) {
        // Stores key-value pairs (Keys must be unique)
        Map<String, Integer> map = new HashMap<>();
        map.put("Alice", 25);
        map.put("Bob", 30);

        System.out.println("Alice's age: " + map.get("Alice"));
    }
}

7. Generics

Generics allow you to create classes, interfaces, and methods that operate on types specified as parameters, improving type safety and preventing runtime errors.

ðŸ‘‰ Generic Class


class Box<T> { // T is a Type Parameter
    private T value;

    public void set(T value) {
        this.value = value;
    }

    public T get() {
        return value;
    }

    public static void main(String[] args) {
        Box<Integer> intBox = new Box<>();
        intBox.set(123);
        // intBox.set("Error"); // Compile-time error, ensuring type safety

        Box<String> strBox = new Box<>();
        strBox.set("Java");
        System.out.println("Value: " + strBox.get());
    }
}

8. File Handling

Java uses the `java.io` package for I/O operations, including reading and writing files. We typically use classes like `FileWriter` and `Scanner`.

ðŸ‘‰ Writing and Reading Files (Conceptual)


import java.io.File;
import java.io.FileWriter;
import java.io.IOException;
import java.util.Scanner;

public class FileDemo {
    public static void main(String[] args) {
        String filename = "demo.txt";

        // 1. Write to File
        try (FileWriter writer = new FileWriter(filename)) {
            writer.write("Intermediate Java concepts.\n");
            writer.write("File handling is key.");
            System.out.println("Wrote data to " + filename);
        } catch (IOException e) {
            System.out.println("Write Error: " + e.getMessage());
        }

        // 2. Read from File
        try {
            File file = new File(filename);
            Scanner reader = new Scanner(file);
            System.out.println("\nReading contents:");
            while (reader.hasNextLine()) {
                System.out.println("-> " + reader.nextLine());
            }
            reader.close();
        } catch (Exception e) {
            System.out.println("Read Error: File not found or read issue.");
        }
    }
}

9. Java 8 Features (Introduction)

Java 8 revolutionized the language with features that support functional programming, including Lambda Expressions and the Stream API.

ðŸ‘‰ Lambda Expressions

A concise way to represent an anonymous function. They are used primarily with Functional Interfaces (interfaces with only one abstract method).


interface Greeting {
    void sayHello(String name);
}

public class MainLambda {
    public static void main(String[] args) {
        // Lambda expression (parameters) -> { body }
        Greeting g = (name) -> System.out.println("Hello " + name + "!");
        g.sayHello("Functional World");
    }
}

ðŸ‘‰ Stream API Basics

Streams enable functional-style operations on sequences of elements (like collections), allowing for powerful data processing in parallel or sequentially.


import java.util.Arrays;
import java.util.List;

public class MainStream {
    public static void main(String[] args) {
        List<String> names = Arrays.asList("Alice", "Bob", "Charlie", "David");

        // Pipeline: source -> intermediate operations -> terminal operation
        names.stream()
             .filter(name -> name.startsWith("C")) // Intermediate: keeps names starting with 'C'
             .map(String::toUpperCase)             // Intermediate: converts to uppercase
             .forEach(System.out::println);        // Terminal: prints the result
    }
}
