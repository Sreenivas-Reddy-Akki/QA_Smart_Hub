Selenium Complete Concepts for Beginners

1. Introduction to Selenium

Selenium is an open-source tool for automating web browsers. For more details, you can visit the official Selenium website. It supports multiple programming languages (like Java, Python, C#, etc.) and browsers (Chrome, Firefox, Edge, Safari).

Key Features:

  • Automates browsers, not desktop apps.
  • Supports multiple browsers and platforms.
  • Integrates with CI/CD tools and frameworks.

Python Example:

from selenium import webdriver

driver = webdriver.Chrome()
driver.get("https://www.google.com")
print(driver.title)
driver.quit()

Java Example:

import org.openqa.selenium.WebDriver;
import org.openqa.selenium.chrome.ChromeDriver;

public class FirstTest {
    public static void main(String[] args) {
        WebDriver driver = new ChromeDriver();
        driver.get("https://www.google.com");
        System.out.println(driver.getTitle());
        driver.quit();
    }
}

2. Selenium Components

Selenium consists of four main components:

Component Description
Selenium IDE Record and playback tool for quick test creation.
Selenium RC (Deprecated) Older version of Selenium, replaced by WebDriver.
Selenium WebDriver Core component to control browsers programmatically.
Selenium Grid Run tests in parallel on multiple machines.

4. Locators in Selenium

Locators are used to identify web elements.

Types of Locators:

  • ID
  • Name
  • Class Name
  • Tag Name
  • Link Text
  • Partial Link Text
  • CSS Selector
  • XPath

Python Example:

driver.find_element("id", "username").send_keys("admin")
driver.find_element("name", "password").send_keys("pass123")
driver.find_element("xpath", "//button[@type='submit']").click()

Java Example:

driver.findElement(By.id("username")).sendKeys("admin");
driver.findElement(By.name("password")).sendKeys("pass123");
driver.findElement(By.xpath("//button[@type='submit']")).click();

5. WebDriver Commands

Basic commands to interact with the browser.

Command Description
get() Opens a URL
getTitle() Gets page title
getCurrentUrl() Gets current URL
close() Closes current tab
quit() Closes browser completely

Python Example:

driver.get("https://example.com")
print(driver.title)
print(driver.current_url)
driver.close()
driver.quit()

Java Example:

driver.get("https://example.com");
System.out.println(driver.getTitle());
System.out.println(driver.getCurrentUrl());
driver.close();
driver.quit();

6. Handling Web Elements

Interacting with input boxes, buttons, checkboxes, dropdowns, etc. using commands like sendKeys() and the Select class.

Python Example:

from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import Select

driver.find_element(By.ID, "username").send_keys("user")
driver.find_element(By.ID, "login").click()
select = Select(driver.find_element(By.ID, "country"))
select.select_by_visible_text("India")

Java Example:

WebElement username = driver.findElement(By.id("username"));
username.sendKeys("user");
driver.findElement(By.id("login")).click();
Select country = new Select(driver.findElement(By.id("country")));
country.selectByVisibleText("India");

7. Waits in Selenium

Used to synchronize test execution with the web page loading.

Types of Waits:

  • Implicit Wait
  • Explicit Wait
  • Fluent Wait

Python Example:

# Implicit Wait
driver.implicitly_wait(10)

# Explicit Wait
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC

wait = WebDriverWait(driver, 10)
element = wait.until(EC.visibility_of_element_located((By.ID, "welcome")))

Java Example:

// Implicit Wait
driver.manage().timeouts().implicitlyWait(Duration.ofSeconds(10));

// Explicit Wait
WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(10));
WebElement element = wait.until(ExpectedConditions.visibilityOfElementLocated(By.id("welcome")));

8. Handling Alerts

Alerts are popup messages on the browser. You must switch to them to interact.

Python Example:

alert = driver.switch_to.alert
print(alert.text)
alert.accept() ¬†# or alert.dismiss()

Java Example:

Alert alert = driver.switchTo().alert();
System.out.println(alert.getText());
alert.accept(); // or alert.dismiss();

9. Handling Frames and Windows

Python Example:

# Switch to frame by name
driver.switch_to.frame("frameName")
driver.switch_to.default_content() # Go back to main page

# Switch to window
handles = driver.window_handles
driver.switch_to.window(handles[1])

Java Example:

driver.switchTo().frame("frameName");
driver.switchTo().defaultContent();

Set<String> handles = driver.getWindowHandles();
for (String handle : handles) {
    driver.switchTo().window(handle);
}

10. Handling Mouse and Keyboard Actions

The ActionChains class is used for complex interactions like drag-drop, hover, right-click, and double-click.

Python Example:

from selenium.webdriver import ActionChains

actions = ActionChains(driver)
element = driver.find_element(By.ID, "drag")
target = driver.find_element(By.ID, "drop")

actions.drag_and_drop(element, target).perform()

Java Example:

Actions actions = new Actions(driver);
WebElement source = driver.findElement(By.id("drag"));
WebElement target = driver.findElement(By.id("drop"));
actions.dragAndDrop(source, target).perform();

12. Data-Driven Testing

Fetching test data from files like Excel or CSV to run the same test with different inputs.

Python Example (using CSV):

import csv
with open('data.csv') as f:
    reader = csv.reader(f)
    for row in reader:
        print(row)

Java Example (using Excel & Apache POI):

FileInputStream file = new FileInputStream("data.xlsx");
XSSFWorkbook workbook = new XSSFWorkbook(file);
XSSFSheet sheet = workbook.getSheetAt(0);
System.out.println(sheet.getRow(0).getCell(0).getStringCellValue());
workbook.close();

13. Selenium Grid

Used for running tests in parallel across multiple browsers and machines, dramatically reducing test execution time.

Key Concepts:

  • Hub: Central point controlling tests.
  • Node: Machines running tests.

Configure using:

java -jar selenium-server-4.x.jar hub
java -jar selenium-server-4.x.jar node --hub http://localhost:4444

14. Page Object Model (POM)

A design pattern to separate test logic from page elements (Locators), making tests maintainable.

Python Example:

class LoginPage:
    def __init__(self, driver):
        self.driver = driver
        self.username = "username"
        self.password = "password"
        self.login_btn = "login"

    def login(self, user, pwd):
        self.driver.find_element(By.ID, self.username).send_keys(user)
        self.driver.find_element(By.ID, self.password).send_keys(pwd)
        self.driver.find_element(By.ID, self.login_btn).click()

Java Example:

public class LoginPage {
    WebDriver driver;
    By username = By.id("username");
    By password = By.id("password");
    By loginBtn = By.id("login");

    public LoginPage(WebDriver driver) {
        this.driver = driver;
    }

    public void login(String user, String pass) {
        driver.findElement(username).sendKeys(user);
        driver.findElement(password).sendKeys(pass);
        driver.findElement(loginBtn).click();
    }
}

15. Test Framework Integration

Integrating Selenium with professional test frameworks for reporting and organization.

Frameworks:

  • Python: pytest or unittest
  • Java: TestNG or JUnit

Python Example (pytest):

def test_google_title(driver):
    driver.get("https://www.google.com")
    assert "Google" in driver.title

Java Example (TestNG):

@Test
public void testGoogleTitle() {
    driver.get("https://www.google.com");
    Assert.assertTrue(driver.getTitle().contains("Google"));
}
