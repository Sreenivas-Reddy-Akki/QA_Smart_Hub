Selenium Learning Guide

1. Introduction to Selenium

What is Selenium?

Selenium is an open-source framework for automating web browsers. It supports multiple programming languages (Java, Python, C#, etc.) and browsers (Chrome, Firefox, Edge, etc.). Selenium is widely used for:

  • Web application testing
  • Web scraping
  • Automating repetitive tasks

Why Learn Selenium?

  • High demand in QA and automation roles
  • Works with multiple programming languages
  • Supports cross-browser testing
  • Integrates with CI/CD pipelines

2. Prerequisites for Selenium

Programming Language

Basic knowledge of Java, Python, or C# (Python is recommended for beginners).

Web Technologies

Understanding of HTML, CSS, and JavaScript (to locate elements on a webpage).

Development Environment

Install an IDE (e.g., PyCharm, VS Code, or Eclipse).

Web Browsers

Install the latest versions of Chrome, Firefox, or Edge.

3. Selenium Installation Process

Step 1: Install Python

Download and install Python from python.org.

Verify installation:

python --version

Step 2: Install Selenium Library

Use pip to install Selenium:

pip install selenium

Step 3: Download WebDriver

Download the appropriate WebDriver for your browser:

  • ChromeDriver
  • GeckoDriver (Firefox)
  • EdgeDriver

Place the WebDriver in a known directory or add it to your system's PATH.

4. Selenium Basics

Launching a Browser

from selenium import webdriver

# Launch Chrome
driver = webdriver.Chrome(executable_path="path/to/chromedriver")

# Open a website
driver.get("https://www.google.com")

Locating Elements

Selenium provides methods to locate elements using:

  • ID
  • Name
  • Class Name
  • CSS Selector
  • XPath
search_box = driver.find_element("name", "q")
search_box.send_keys("Selenium")
search_box.submit()

5. Intermediate Selenium Topics

Handling Waits

  • Implicit Wait: Waits for a specified time before throwing an exception.
  • Explicit Wait: Waits for a specific condition.
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC

element = WebDriverWait(driver, 10).until(
    EC.presence_of_element_located((By.ID, "element_id"))
)

Handling Alerts and Popups

alert = driver.switch_to.alert
alert.accept()  # Clicks "OK"
alert.dismiss() # Clicks "Cancel"

Working with Frames

driver.switch_to.frame("frame_name")  # Switch to frame
driver.switch_to.default_content()    # Switch back to main page

6. Advanced Selenium Topics

Page Object Model (POM)

A design pattern to create an object repository for web elements.

class LoginPage:
    def __init__(self, driver):
        self.driver = driver
        self.username_field = (By.ID, "username")
        self.password_field = (By.ID, "password")
        self.login_button = (By.ID, "login")

    def login(self, username, password):
        self.driver.find_element(*self.username_field).send_keys(username)
        self.driver.find_element(*self.password_field).send_keys(password)
        self.driver.find_element(*self.login_button).click()

Headless Browsing

Run tests without opening a browser window.

from selenium.webdriver.chrome.options import Options

options = Options()
options.add_argument("--headless")
driver = webdriver.Chrome(options=options)

Cross-Browser Testing

Run the same test on multiple browsers.

browsers = ["chrome", "firefox", "edge"]
for browser in browsers:
    if browser == "chrome":
        driver = webdriver.Chrome()
    elif browser == "firefox":
        driver = webdriver.Firefox()
    elif browser == "edge":
        driver = webdriver.Edge()
    driver.get("https://www.google.com")
    print(f"Title in {browser}: {driver.title}")
    driver.quit()

7. Selenium with Test Frameworks

UnitTest (Python)

import unittest
from selenium import webdriver

class TestGoogleSearch(unittest.TestCase):
    def setUp(self):
        self.driver = webdriver.Chrome()

    def test_search(self):
        self.driver.get("https://www.google.com")
        search_box = self.driver.find_element("name", "q")
        search_box.send_keys("Selenium")
        search_box.submit()
        assert "Selenium" in self.driver.title

    def tearDown(self):
        self.driver.quit()

if __name__ == "__main__":
    unittest.main()

Pytest

import pytest
from selenium import webdriver

@pytest.fixture
def driver():
    driver = webdriver.Chrome()
    yield driver
    driver.quit()

def test_google_search(driver):
    driver.get("https://www.google.com")
    search_box = driver.find_element("name", "q")
    search_box.send_keys("Selenium")
    search_box.submit()
    assert "Selenium" in driver.title

8. Best Practices

  • Use Page Object Model (POM) for maintainability.
  • Implement waits to handle dynamic elements.
  • Run tests in headless mode for faster execution.
  • Use CI/CD integration (e.g., Jenkins, GitHub Actions) for automated testing.
